APPROACH:
  - Two-layer word list: 77 multi-word phrases (exact match) + 23 single words (word boundary regex)
  - Categories: financial promises, urgency/pressure, too-good-to-be-true, generic sales, deceptive/phishing
  - scanForSpamWords(subject, body) → { matchCount, matches, flagged }
  - Integrated in draftWithQualityGate(): scans BEFORE LLM scoring (instant, zero cost)
  - If flagged (≥3 matches): adds specific issue to quality score, penalizes overall by -1, forces retry
  - Email only passes quality gate if score ≥7 AND not spam-flagged
  - Tests cover: clean emails, spammy emails, case insensitivity, word boundaries, edge cases, 6 real-world scenarios
LEARNINGS:
  - Word boundary matching (\b) prevents false positives (e.g. "discovery" doesn't match "disco")
  - Multi-word phrases checked first to avoid double-counting (e.g. "act now" is one match, not "act" + "now")
  - Showing matches in the issue helps the LLM understand what to fix on retry
  - Context-aware: well-written cold emails pass with 0 matches; only genuine spam patterns trigger
